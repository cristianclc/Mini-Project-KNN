Reclasificación de la Variable Objetivo#

Al observar la distribución de la variable Target, se identifican tres categorías: Dropout (abandono), Enrolled (matriculado) y Graduate (graduado). Sin embargo, el objetivo principal de este proyecto es predecir si un estudiante abandona o no abandona la institución, es decir, un problema de clasificación binaria.

Por lo tanto, se decide reclasificar los registros con estado “Enrolled” como “Graduate” (no abandono). Esta decisión se fundamenta en dos razones:

  1. Coherencia con el objetivo del proyecto: Se busca identificar estudiantes en riesgo de deserción, por lo que la clase de interés es Dropout, mientras que todas las situaciones de permanencia (graduados y aún matriculados) se agrupan como “no abandono”.

  2. Alineación con la documentación del dataset: Según el artículo introductorio de la base de datos UCI, las categorías Graduate y Enrolled representan diferentes niveles de desempeño de estudiantes que no abandonaron la institución. Por tanto, su agrupación es conceptualmente válida.

Esta transformación reduce el problema a dos categorías claramente definidas (retención vs. deserción), lo que facilita tanto el modelado con KNN (evitando clasificación multiclase) como la interpretación de métricas como precisión, sensibilidad y especificidad.

Reducción de Categorías en Variables de Alta Cardinalidad#

Al analizar las variables categóricas, se identifica que algunas presentan una alta cardinalidad (un número elevado de categorías únicas). Muchas de estas categorías cuentan con muy pocas observaciones (incluso una sola), lo que puede generar dos problemas para el modelo KNN:

  1. Sobreajuste (overfitting): Categorías muy poco frecuentes pueden hacer que el modelo aprenda patrones espurios que no se generalizan.

  2. Inestabilidad en codificación: Al aplicar One-Hot Encoding, categorías raras crean columnas con escasos valores positivos, introduciendo ruido en lugar de información útil.

Para mitigar estos problemas, se opta por agrupar todas las categorías con baja frecuencia en una sola etiqueta denominada “Otras”. Se establece un umbral mínimo de 15 observaciones por categoría para mantener su identidad original. Las categorías que no alcanzan este umbral se reclasifican como 'Otras'.

for i in variables_categoricas:

Selección de Variables Categóricas mediante Prueba Chi-Cuadrado#

Para evaluar la relevancia de las variables categóricas en la predicción del abandono, se aplica la prueba de independencia Chi-Cuadrado (χ²). Esta prueba estadística permite determinar si existe una asociación significativa entre cada variable categórica y la variable objetivo y (codificada como 0 = Graduado, 1 = Abandono).

Interpretación de la prueba:

  • Hipótesis nula (H₀): La variable categórica es independiente de la variable objetivo (no hay relación).

  • Hipótesis alternativa (H₁): Existe una relación significativa entre ambas variables.

Criterio de selección:

  • Un valor p < 0.05 indica que la relación es estadísticamente significativa.

  • Un estadístico χ² más alto sugiere una asociación más fuerte con la variable objetivo.

Este análisis ayuda a:

  1. Identificar qué variables categóricas son más relevantes para predecir el abandono.

  2. Detectar variables redundantes o irrelevantes que podrían eliminarse para simplificar el modelo KNN.

Selección de Variables Numéricas mediante Prueba de Kruskal-Wallis#

Para evaluar la relación entre las variables numéricas y la variable objetivo categórica Target (0 = Graduado, 1 = Abandono), se aplica la prueba no paramétrica de Kruskal-Wallis.

¿Por qué Kruskal-Wallis y no ANOVA? A diferencia del ANOVA, la prueba de Kruskal-Wallis no requiere los supuestos de normalidad ni de igualdad de varianzas entre los grupos. Esto es especialmente relevante en este dataset, donde como se observó en el análisis exploratorio, varias variables numéricas presentan asimetrías pronunciadas, valores atípicos y distribuciones no normales.

Interpretación de la prueba:

  • Hipótesis nula (H₀): Las distribuciones de la variable numérica son idénticas entre las categorías de Target (no hay diferencia).

  • Hipótesis alternativa (H₁): Al menos una categoría de Target tiene una distribución diferente.

Criterio de selección:

  • Un valor p < 0.05 indica diferencias estadísticamente significativas entre los grupos.

  • Un estadístico H más alto sugiere una mayor capacidad discriminante de la variable numérica.

Este análisis permite:

  1. Identificar qué variables numéricas diferencian significativamente entre estudiantes que abandonan y los que no.

  2. Priorizar las variables con mayor poder predictivo para el modelo KNN.

Tratamiento de Variables Categóricas para el Modelo de Clasificación#

Para preparar los datos para el modelo KNN de clasificación, es necesario transformar las variables categóricas en un formato numérico que el algoritmo pueda procesar. Se aplica la técnica de One-Hot Encoding, la cual crea una nueva columna binaria (0 o 1) por cada categoría única presente en las variables originales.

Variables a codificar:

  • Application mode (modo de solicitud de ingreso)

  • Course (curso o programa de estudios)

¿Por qué One-Hot Encoding?

  • KNN calcula distancias entre observaciones; un encoding ordinal (asignar números 1,2,3…) introduciría relaciones de orden inexistentes entre categorías.

  • One-Hot Encoding representa cada categoría de manera independiente, evitando que el modelo asuma jerarquías falsas.

Detalles técnicos:

  • sparse=False → devuelve una matriz densa en lugar de dispersa.

  • drop=None → mantiene todas las categorías (sin eliminar la primera).

  • handle_unknown="ignore" → si durante la predicción aparece una categoría no vista en entrenamiento, se asigna todo ceros.

Las variables Application mode y Course se eliminan del DataFrame original y se reemplazan por las nuevas columnas codificadas. Las demás variables (incluyendo las categóricas binarias como Tuition fees up to date, Scholarship holder y Debtor) ya están en formato numérico (0/1) y no requieren transformación adicional.

ohe = OneHotEncoder(sparse=False, drop=None, handle_unknown="ignore")

Tratamiento de Variables Numéricas para el Modelo de Clasificación#

Las variables numéricas seleccionadas presentan diferentes escalas y rangos de valores. Dado que el algoritmo KNN se basa en el cálculo de distancias (generalmente distancia euclidiana), es fundamental escalar las variables para que todas contribuyan equitativamente al modelo. De lo contrario, las variables con rangos más amplios dominarían la medida de distancia.

Elección del escalador: RobustScaler

Se utiliza RobustScaler en lugar de StandardScaler o MinMaxScaler por las siguientes razones:

  • Robustez frente a valores atípicos (outliers): A diferencia de StandardScaler (que usa media y desviación estándar), RobustScaler emplea la mediana y el rango intercuartílico (IQR) , estadísticos resistentes a outliers.

  • Interpretación del escalamiento:

    • La mediana se transforma a 0

    • El valor correspondiente al Q3 (tercer cuartil) se aproxima a 0.5

    • El valor correspondiente al Q1 (primer cuartil) se aproxima a -0.5

Esta transformación preserva la escala relativa de las variables mientras mitiga el impacto de valores atípicos extremos, que son comunes en variables como el número de unidades aprobadas o las calificaciones.

Aplicación de SMOTE#

Para tratar el desbalance de clases, se utiliza SMOTE (Synthetic Minority Over-sampling TEchnique) . Esta técnica:

  • Genera muestras sintéticas de la clase minoritaria en lugar de simplemente duplicar ejemplos existentes.

  • Funcionamiento: Para cada muestra de la clase minoritaria, SMOTE selecciona uno o más de sus vecinos más cercanos (dentro del espacio de características) y crea nuevas muestras interpolando entre ellos.

  • Ventaja sobre el sobremuestreo simple: Reduce el riesgo de sobreajuste, ya que introduce nueva información sintética en lugar de replicar ejemplos idénticos.

Configuración aplicada:

  • random_state=1 → fija la semilla para garantizar reproducibilidad.

  • El balanceo se aplica solo sobre el conjunto de entrenamiento, no sobre validación o prueba, para evitar fugas de información.

Proceso de Validación Cruzada#

Para cada valor de k:

  1. Se realizan 5 iteraciones (folds) con el conjunto de entrenamiento balanceado.

  2. En cada iteración, se entrena un clasificador KNN con un subconjunto diferente (train fold) y se evalúa sobre el fold de validación.

  3. Se calcula la curva ROC y el AUC para cada fold.

  4. Se promedia la curva ROC y el AUC a través de los 5 folds, obteniendo:

    • mean_auc → AUC promedio

    • mean_tpr → Tasa de Verdaderos Positivos promedio para graficar la curva ROC promedio

El mejor valor de k se selecciona como aquel que maximiza el AUC promedio en la validación cruzada. Los resultados (falsos positivos, verdaderos positivos y AUC) se almacenan para cada k con fines de visualización posterior.

Selección de Variables Categóricas para Regresión mediante Kruskal-Wallis#

Para determinar qué variables categóricas tienen mayor relevancia en la predicción del promedio final (variable objetivo continua y_reg), se aplica nuevamente la prueba no paramétrica de Kruskal-Wallis.

Objetivo: Evaluar si existen diferencias estadísticamente significativas entre las medianas del promedio final (y_reg) a través de las diferentes categorías de cada variable categórica.

¿Por qué Kruskal-Wallis y no ANOVA?

  • La prueba de Kruskal-Wallis no requiere normalidad ni igualdad de varianzas, lo cual es adecuado dado que y_reg puede no seguir una distribución normal.

  • Esta prueba compara medianas en lugar de medias, siendo más robusta frente a valores atípicos.

Interpretación de la prueba:

  • Hipótesis nula (H₀): Las distribuciones de y_reg son idénticas entre las categorías de la variable (no hay diferencia).

  • Hipótesis alternativa (H₁): Al menos una categoría tiene una distribución diferente de y_reg.

Criterio de selección:

  • Un valor p < 0.05 indica que la variable categórica está asociada significativamente con el promedio final.

  • Un estadístico H más alto sugiere una mayor capacidad discriminante de la variable categórica respecto al rendimiento académico.

Este análisis permite:

  1. Identificar qué variables categóricas influyen significativamente en el promedio final.

  2. Priorizar aquellas con mayor poder predictivo para el modelo KNN de regresión.

Selección de Variables Numéricas para Regresión mediante Correlación de Spearman#

Para evaluar la relación entre las variables numéricas y la variable objetivo continua (y_reg = promedio final), se utiliza el coeficiente de correlación de Spearman.

¿Por qué Spearman y no Pearson?

  • No requiere supuestos de normalidad: A diferencia de Pearson, Spearman no asume que los datos sigan una distribución normal.

  • Robusto frente a valores atípicos (outliers): Spearman trabaja con rangos (orden) en lugar de valores brutos, lo que lo hace resistente a outliers.

  • Captura relaciones monótonas (no solo lineales): Detecta si cuando una variable aumenta, la otra tiende a aumentar o disminuir, incluso si la relación no es estrictamente lineal.

  • Útil con distribuciones asimétricas o con colas pesadas: Condiciones presentes en varias variables numéricas del dataset.

Interpretación del coeficiente de Spearman:

  • Corr > 0: Relación positiva – a mayor valor de la variable, mayor promedio final.

  • Corr < 0: Relación negativa – a mayor valor de la variable, menor promedio final.

  • |Corr| cercano a 1: Relación monótona fuerte.

  • |Corr| cercano a 0: Relación débil o nula.

Criterio de selección:

  • Un valor p < 0.05 indica que la correlación es estadísticamente significativa.

  • A mayor valor absoluto de la correlación (|Corr|), mayor poder predictivo potencial.

  1. Identificar qué variables numéricas están más fuertemente asociadas con el promedio final.

  2. Priorizar aquellas con mayor capacidad predictiva para el modelo KNN de regresión.

Al igual que en el modelo de clasificación, las variables categóricas no dicotómicas deben transformarse a formato numérico para que el algoritmo KNN pueda procesarlas correctamente. Se aplica nuevamente la técnica de One-Hot Encoding sobre las variables categóricas seleccionadas.

Variables a codificar:

  • Course (curso o programa de estudios)

¿Por qué One-Hot Encoding?

  • El KNN calcula distancias entre observaciones; asignar valores ordinales (1, 2, 3…) a las categorías introduciría relaciones de orden inexistentes.

  • One-Hot Encoding crea una columna binaria independiente para cada categoría, permitiendo que el modelo trate cada modalidad como un atributo separado sin jerarquías falsas.

Detalles técnicos:

  • sparse=False → devuelve una matriz densa en lugar de dispersa.

  • drop=None → mantiene todas las categorías (sin eliminar la primera).

  • handle_unknown="ignore" → si durante la predicción aparece una categoría no vista en entrenamiento, se asigna todo ceros.

Proceso:

  1. Se aplica One-Hot Encoding a las columnas Application mode y Course.

  2. Se generan nuevas columnas binarias (una por cada categoría única).

  3. Se eliminan las columnas originales del DataFrame.

  4. Se concatenan las nuevas columnas codificadas.

La variable categórica Tuition fees up to date ya es binaria (0/1) y no requiere transformación adicional.

Tratamiento de Variables Numéricas para el Modelo de Regresión#

Al igual que en el modelo de clasificación, las variables numéricas seleccionadas presentan diferentes escalas y rangos de valores. Dado que el algoritmo KNN se basa en el cálculo de distancias (generalmente distancia euclidiana), es fundamental escalar las variables para que todas contribuyan equitativamente al modelo.

Elección del escalador: RobustScaler

Se utiliza RobustScaler por las siguientes razones:

  • Robustez frente a valores atípicos (outliers): A diferencia de StandardScaler (que usa media y desviación estándar), RobustScaler emplea la mediana y el rango intercuartílico (IQR) , estadísticos resistentes a outliers.

  • Preservación de la escala relativa: La transformación mantiene la relación entre los valores típicos de cada variable.

  • Interpretación del escalamiento:

    • La mediana se transforma a 0

    • El valor correspondiente al Q3 (tercer cuartil) se aproxima a 0.5

    • El valor correspondiente al Q1 (primer cuartil) se aproxima a -0.5

Selección Manual del Mejor k mediante Validación Cruzada (Regresión)#

Siguiendo las restricciones didácticas del proyecto (no se permite el uso de GridSearchCV), se implementa una búsqueda manual de hiperparámetros para identificar el valor óptimo del número de vecinos (k) en el modelo KNN de regresión.

Configuración de la búsqueda:

Parámetro

Valor(es)

Justificación

Valores de k evaluados

1 a 21 (todos los enteros)

Se explora un rango amplio para identificar el punto óptimo donde se minimiza el error

Número de folds (K-Fold)

5

Cada modelo se evalúa en 5 particiones diferentes; el rendimiento final es el promedio de los 5 folds

Métrica de evaluación principal

RMSLE (Root Mean Squared Logarithmic Error)

Mide el error porcentual entre valores predichos y reales; es menos sensible a outliers que RMSE y adecuado para variables positivas como calificaciones

¿Por qué RMSLE en lugar de RMSE o MAE?

  • Las calificaciones (y_reg) son variables positivas.

  • RMSLE penaliza errores relativos en lugar de absolutos: predecir 12 en lugar de 10 tiene similar impacto que predecir 2 en lugar de 0.

  • Es robusto frente a valores atípicos (outliers), ya que aplica logaritmo antes de calcular el error.

Fórmula del RMSLE:

RMSLE = √( (1/n) * Σ (log(y_pred + 1) - log(y_true + 1))² )

Proceso de validación cruzada:

Para cada valor de k (1 a 21):

  1. Se realizan 5 iteraciones (folds) con el conjunto de entrenamiento.

  2. En cada iteración, se entrena un regresor KNN con un subconjunto diferente (train fold) y se evalúa sobre el fold de validación.

  3. Se calcula el RMSLE para cada fold.

  4. Se promedia el RMSLE a través de los 5 folds.

Selección del mejor parámetro: El mejor valor de k es aquel que minimiza el RMSLE promedio en la validación cruzada.
